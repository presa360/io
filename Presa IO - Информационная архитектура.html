01

Инвентарь контента

Всё, что пользователь должен найти, сделать или понять — сгруппировано по частоте использования. Частота определяет, что должно быть в один клик, а что можно спрятать глубже.

Каждую сессиюCORE
  • Создать презентацию — из текста или с нуля
  • Вставить / импортировать текст в Prompt Manager
  • Подтвердить структуру — сколько слайдов и какие
  • Редактировать текст слайда прямо на холсте
  • Сменить вариант дизайна слайда
  • Добавить слайд из библиотеки
  • Предпросмотр презентации
  • Скачать — PPTX / PDF
РегулярноSUPPORT
  • Выбрать стиль презентации
  • Выбрать шаблон слайда по категории
  • Переставить / удалить слайды
  • Заменить фото в фото-слоте
  • Выбрать иконку (поиск по библиотеке)
  • Переименовать презентацию
  • Отменить / повторить действие
  • AI-сборка одного слайда из текста
Изредка / разовоSETUP
  • Настроить бренд — логотип, акцент, название
  • Скачать фирменные шрифты (для PPTX)
  • Открыть сохранённую презентацию
  • Сменить тему / палитру
  • Понять, как продукт работает (лендинг)
  • Настройки аккаунта
Принцип: «Core» — всегда в один клик и в основной рабочей зоне; «Support» — в панелях и контекстных меню; «Setup» — за вторичным входом, не мешает основному потоку.

04

Предполагаемые точки трения

4 момента с наибольшей вероятностью отказа — где пользователь закрывает вкладку, не дойдя до цели.

1
«Сколько слайдов получится?»
ПОТОК A · Prompt Manager → Структура
Пользователь вставил текст и не уверен, что система разберёт его правильно — раньше 6 разделов превращались в 10, и счётчик врал. Плюс развилка «весь текст / сжать» заставляет думать.
Честный счётчик + предпросмотр структуры до генерации (уже улучшено); подсказать рекомендованный режим по умолчанию.
2
Паралич выбора стиля
ПОТОК B · Выбрать стиль
Без понимания разницы между стилями пользователь боится выбрать не то и застревает. Слишком много вариантов раньше усиливали ступор.
Сокращено до 3 стилей + «Чистый лист»; добавить превью-миниатюры и пометку «Популярный».
3
Шрифт «слетел» в PPTX
ПОТОК A/E · Скачать → открыть в PowerPoint
На самом финише пользователь видит, что презентация выглядит иначе, чем в редакторе — нет фирменного шрифта на ПК. Разочарование в точке наибольшей ценности.
Кнопка «Фирменные шрифты» + подсказка после экспорта (сделано); повысить заметность подсказки.
4
Пустой холст «с чего начать»
ПОТОК C · Чистый лист → Редактор
После выбора «Чистый лист» пользователь видит пустой редактор и не знает следующего шага — нужно догадаться про библиотеку слайдов.
Сразу открывать библиотеку «Добавить слайд» (сделано); добавить пустое состояние с подсказкой-стрелкой.

07

Приоритизация по влиянию

Что проектировать первым — по объёму использования × влиянию на бизнес-цель (созданная и скачанная презентация).

P0
Prompt Manager + предпросмотр структуры
Ядро ценности и главный источник отказов. Если разбивка неверна — теряется всё остальное.
Объём Влияние Статус: улучшено
P0
Редактор — холст + Контент/Дизайн
Используется каждую сессию. Здесь пользователь проводит 80% времени.
Объём Влияние Статус: активно
P1
Экспорт PPTX/PDF + фирменные шрифты
Точка достижения бизнес-цели. Провал здесь обнуляет всю работу пользователя.
Объём Влияние Статус: сделано
P1
Библиотека слайдов + AI-сборка
Двигатель потоков C и D. Чем легче найти и собрать слайд — тем выше завершаемость.
Объём Влияние Статус: в работе
P2
Выбор стиля + Мои презентации
Важны, но используются реже и не на каждом пути. Влияют на возврат и первое впечатление.
Объём Влияние Статус: базово есть
P3
Бренд-сетап + Настройки
Разовые действия. Нужны, но не блокируют основной путь к цели.
Объём Влияние Статус: позже
Порядок проектирования: сначала довести P0 (Prompt Manager → Редактор) до безупречности, затем закрепить P1 (экспорт + библиотека), и только потом полировать P2–P3.
