## A. ХОЛСТ (canvas) — рендер слайдов

### A1 — P0: контент вылезает за кадр 16:9 (теряются данные)
Замеры — доля переполнения по низу кадра (slide = 258×145 в пробе, пропорции держатся на любом размере):

| Шаблон | layoutType | Переполнение | Что вылезает |
|---|---|---|---|
| **numbers-06** | (numbers) | **+75px / 52%** | блок `stats` — сетка цифр слишком высокая |
| **con-services** | con_services | **+35px / 24%** | карточки процесса 01–05 (construction pack) |
| **services-08** | (services) | **+33px / 23%** | карточки + `panelText` |
| services-14 | service_compare | +19px / 13% | сравнительная `<table>` выше кадра |
| comparison-04 | (comparison) | +13px / 9% | матрица колонок |
| numbers-07 | (numbers) | +8px / 6% | мелкое, по низу |

**Корневые причины (одни и те же):**
1. число элементов не ограничено (`slice` отсутствует или великоват) для плотного контента;
2. фикс. `gap`/`padding` не сжимаются при 4–6 элементах;
3. контейнер не получает `min-height:0` + `justify-content:center`, поэтому стек выше кадра;
4. `fit.js → computeScale` не покрывает эти layoutType своей «плотностной» логикой (не отдаёт
   `scale < 1`), поэтому AutoFitText ужимает текст построчно, но **высоту стопки не лечит**.

**Что делаем (на каждый из 6):**
- cap по количеству (`.slice(0, N)`), адаптивные `gap`/`fontSize` от числа элементов
  (как уже сделано в `L_numbersIconStats`/`L_benefitsCheckRings` — взять за образец);
- обёртке контента — `flex:1 1 0; min-height:0; justify-content:center`;
- завести/расширить ветку в `computeScale` для этих layoutType (плотность → `scale`).

### A2 — P1: line‑clamp срабатывает уже на sample
`services-04` (12 обрезок), `benefits-05` (12), `services-05/07/09/12`, `benefits-06`,
`solution-08`, `numbers-04`, `pricing-02/05`, `comparison-01/03`, `team-06`, `title-14/16`.

Это значит: либо `sample`‑текст длиннее, чем влезает в раскладку, либо `-webkit-line-clamp`
слишком жёсткий. **Эталонные данные обязаны влезать без обрезки.**
- по каждому: либо подрезать `sample` в `templates.js` до реального лимита раскладки,
  либо поднять `maxLines` / снизить `baseCqw` в рендере.
- табличные `clipped:1` (ellipsis в ячейке) — штатно, не трогаем.

### A3 — P2: декоративный выход за край — проверить, не баг
`title-11`, `title-15` (концентрические кольца), `contact-04`, `contact-06` (полосы/панели).
Текст не теряется (пустой `textContent` у элементов‑оверфлоу) → вероятно намеренный bleed,
клипается `overflow:hidden`. **Глянуть скриншотом, подтвердить — иначе оставить как есть.**
`chart-08 +7658px` — ложноположительное: это `<defs>/<linearGradient>/<stop>` (невидимы). Игнор.

### B3 — P1: автоматический guard синхронизации
Завести разовый скрипт‑проверку (держать в `prompts/` или как dev‑probe):

Гонять после каждой партии правок — это и есть «экспорт‑чек» из рабочего процесса.
